## 1. 專案總覽

### 1.1 一句話定義

拍照上傳醫療單，AI 幫你秒懂、秒整理、秒提醒。

### 1.2 背景與動機

家人出院後拿到大量衛教單（3 張藍色正反印、2 張黃色正反印、1 張注意事項），資訊量大、術語多、家人記不住甚至記錯。照護者花了一整天用 Claude 整理、ChatGPT 生圖解漫畫、7-11 列印 A3/A4，才做出家人看得懂的海報。還好有 Notion AI 語音紀錄補捉護理站口頭交代的資訊。

CareDoc 要把這個「一整天」變成「3 分鐘」。

### 1.3 專案定位

- POC（概念驗證），先自己用再開源
- 手機原生 App（iOS + Android）
- 早期建立收費點，API 費用由開發者統一負擔
- 做工具本身是照護者消化壓力的方式，與照顧家人不衝突

### 1.4 成功標準

- 自己用比手動流程快
- 吸引前 100 名付費用戶
- GitHub 有人 star / fork

### 1.5 長期承諾

療程結束後持續維護，因為這個問題不會消失，其他家庭也會遇到。

## 2C. 隱私與資料保護

### 核心原則

**CareDoc 不擁有使用者的醫療資料。所有醫療資料僅存在使用者手機，伺服器不碰、不存、不看。**

### 架構：本地優先、伺服器無知

```
使用者手機
├── 照片/文件 → 只存在手機本地
├── AI 整理結果 → 只存在手機本地（expo-sqlite）
├── 快樂花園進度 → 只存在手機本地
└── 所有醫療資料 → 永遠不離開手機

伺服器（Supabase）
├── 只處理：帳號登入（email）、付費狀態、App 設定
├── 不儲存：任何醫療文件、照片、AI 結果
└── Edge Function → 純轉接站，不留紀錄
```

### 各環節隱私設計

| 環節 | 做法 | 碰到個資？ |
|------|------|----------|
| 使用者上傳照片 | 照片只存手機記憶體，不上傳 Storage | ❌ |
| AI 整理（Claude API） | Edge Function 轉接，不記錄 request body | ❌ 過路不留 |
| AI 結果儲存 | expo-sqlite 本地儲存，不同步雲端 | ❌ |
| 藥品資料庫查詢 | 只傳藥名（非個資） | ❌ |
| 帳號系統 | 只存 email + 付費狀態 | ⚠️ 僅 email |
| SNS 試玩版 | 完全無狀態，session 結束全消失 | ❌ |

### Claude API 資料政策

使用者醫療照片經 Claude API 處理。Anthropic API 政策不使用 API 輸入訓練模型，但資料會經過其伺服器。App 內需明確告知使用者。

### 必要法律文件

| 文件 | 內容 | 顯示位置 |
|------|------|---------|
| 隱私政策 | 「不儲存醫療資料，所有資料僅存在手機」 | App 設定 + 官網 |
| 使用者同意書 | 「AI 處理過程資料經第三方 API，處理完即刪除」 | 首次使用時 |
| 免責聲明 | 「AI 整理僅供參考，請與原始醫療文件核對」 | 每次結果頁底部 |

## 3. 商業模式與定價

### 3.1 架構決策

開發者統一負擔 API 費用。使用者不需自備 API Key，所有 AI 呼叫透過 Supabase Edge Functions 走開發者帳號。

### 3.2 計費單位：照護時段

一個「照護時段」= 24 小時內不限上傳張數、不限 AI 整理次數。以「回診」而非「上傳」為計費單位。

| 行為 | 是否扣次 |
|------|---------|
| 同一天內上傳第 1 張文件 | 扣 1 次（開啟時段） |
| 同一天內上傳第 2-10+ 張 | 不扣（同一時段） |
| 同一天內追問補充、更新資料 | 不扣 |
| 隔天上傳新文件 | 扣 1 次（新時段） |
| 查看 App 內行事曆 | 永遠不扣 |
| 手動點「加入手機行事曆」(.ics) | 永遠不扣 |
| 使用快樂花園 | 永遠不扣 |

### 3.3 三層定價

| | 🌱 免費 | 🌿 輕量（儲值） | 🌳 吃到飽（月費） |

| **每時段上傳** | 不限張數 | 不限張數 | 不限張數 |
| **滾動更新** | ❌ 每次獨立 | ✅ | ✅ |
| **App 內行事曆** | ✅ 檢視行程列表 | ✅ 檢視行程列表 | ✅ 檢視行程列表 |

| **白話版注意事項** | 🔒 模糊遮罩 | 🔒 模糊遮罩 | ✅ 月費專屬 |

### 3.4 加購項目

| 項目 | 價格 | RevenueCat 類型 | 說明 |
|------|------|----------------|------|
| 📋 照護海報匯出 | $49 台幣 買斷 | Non-consumable | 解鎖完整版海報，可列印 A3/A4，含 DALL-E 生圖 |
| 🔔 行事曆推播提醒 | $29 台幣/月 | Auto-renewable | 回診前一天+前一小時推播、每日用藥提醒 |

**白話版用藥注意事項**不獨立販售，綁在月費 $149 吃到飽方案中，作為升級誘因。免費和輕量版使用者看到白話翻譯會顯示模糊遮罩 +「升級月費解鎖」。

所有收費入口透過 **RevenueCat** 統一追蹤，涵蓋 iOS App Store、Google Play 及未來 Web 儲值。

**RevenueCat 產品設定：**

| 產品 | 類型 | 價格 |
|------|------|------|
| 儲值包 3 時段 | Consumable | $30 |
| 月費吃到飽（含白話版） | Auto-renewable subscription | $149/月 |
| 照護海報匯出 | Non-consumable | $49 |
| 行事曆推播提醒 | Auto-renewable subscription | $29/月 |

**設計理由：**

- **行事曆「看」免費，「提醒」收費**：App 內檢視行程永遠免費，推播提醒才是加購。區分「資訊整理」（核心價值）和「主動服務」（進階價值）
- **海報一次買斷**：高感知價值但低使用頻率，適合一次買斷，也是第一次小額付費的低門檻入口
- **專業版注意事項免費**：用藥安全是基本社會責任，不放付費牆後。白話翻譯是 AI 的附加價值
- **.ics 匯出不受影響**：手動點「加入手機行事曆」永遠免費

### 3.5 轉換路徑

```
免費試用 2 次回診（體驗完整價值）
     ↓ 第 3 次被擋
  「儲值 $30 解鎖 3 次回診」← 多數人選這個
     ↓ 看到海報很好看，想印
  「加購海報 $49 一次買斷」← 第一次小額付費
     ↓ 覺得推播提醒很方便
  「加購提醒 $29/月」← 被動收入
     ↓ 想看白話版注意事項 + 每月買儲值包不划算
  「升級月費 $149 全都有」← 白話版是月費專屬誘因
```

### 3.6 成本與利潤

單次照護時段 API 成本（不含海報，海報為加購項目獨立計算）：約 $2 台幣

| 方案 | 單時段收入 | 成本 | 毛利 |
|------|----------|------|------|
| 輕量 $30/3 | $10/次 | $2 | +$8 (80%) |
| 吃到飽正常用 10 次/月 | $14.9/次 | $2 | +$12.9 (87%) |
| 吃到飽重度用 15 次/月 | $9.9/次 | $2 | +$7.9 (80%) |
| 海報加購 $49 買斷 | $49 | DALL-E ~$1.3/次 | 高利潤 |
| 推播提醒 $29/月 | $29/月 | 推播成本 ~$0 | 幾乎純利 |

### 3.7 公益帳號

不設自動公益池。由管理者手動審核：

- 使用者提供佐證資料（低收入戶證明、重大傷病卡、身心障礙證明等）
- 管理者在 Supabase 後台手動開通，給予免費照護時段額度
- 後台可設定每個公益帳號的額度和有效期限

### 3.8 防濫用

- 吃到飽每月上限 15 次照護時段
- 照護海報（DALL-E）為加購項目，自然控制成本
- 超過上限跳出提示，可聯繫客服加額度

## 4. 🌈 快樂區：設計自己的快樂

> 照顧家人的人，往往忘了照顧自己。快樂區完全免費，不扣任何額度。

### 4.1 設計理念

照護者長期處於高壓狀態，需要隨手可得的減壓工具。快樂區不是附加功能，而是 CareDoc 核心價值的延伸——照顧家人，也照顧自己。

### 4.2 底層理論：六大快樂化學物質

基於神經化學（Neurochemistry of Well-being）框架，人體有六種化學物質共同構成幸福感的完整光譜。快樂區的所有功能都對應到具體的神經化學機制：

| # | 化學物質 | 別名 | 觸發方式 | 對應功能 |
|---|---------|------|---------|---------|
| 1 | 多巴胺 Dopamine | 獎勵分子 | 設定小目標、學新技能 | 每日待辦完成慶祝動畫 |
| 2 | 血清素 Serotonin | 穩定之光 | 曬太陽、規律運動、感恩練習 | 感恩日記、金句提醒 |
| 3 | 催產素 Oxytocin | 連結之環 | 擁抱、眼神交流、深度對話 | 心情小卡分享給家人 |
| 4 | 腦內啡 Endorphin | 天然止痛藥 | 高強度運動、大笑、冥想 | 呼吸練習、微運動 |
| 5 | 睪固酮 Testosterone | 征服之峰 | 贏得小勝利、充足睡眠 | 照護里程碑徽章 |
| 6 | 腎上腺素 Adrenaline | 瞬燃引擎 | 突破恐懼、冷水浴 | 舒適圈小挑戰卡 |

### 4.3 神經可塑性：快樂可以「練」出來

> 「一起放電的神經元，會綁在一起。」— Donald Hebb

三階段迴路養成：
1. **刻意啟動**（1-21 天）：需要有意識行動，設提醒、刻意練習
2. **迴路強化**（21-66 天）：阻力降低，身體自己想做
3. **習慣化快樂**（66 天後）：迴路成為預設路徑，自動運行

App 內建「快樂迴路追蹤器」，記錄使用者的練習天數，視覺化迴路養成進度。

### 4.4 六大功能模組

#### 🫁 減壓呼吸練習（觸發腦內啡 + 血清素）

- **478 呼吸法**：吸氣 4 秒 → 屏氣 7 秒 → 吐氣 8 秒
- **方形呼吸法**：吸 4 秒 → 停 4 秒 → 吐 4 秒 → 停 4 秒
- **漸進式放鬆**：從腳到頭逐步放鬆肌肉群
- 動畫引導圓圈（隨呼吸節奏縮放）+ 柔和音效
- 可設定 1 / 3 / 5 / 10 分鐘

#### 🏃 簡單運動（觸發腦內啡 + 睪固酮）

- 適合在醫院走廊、等候區、家中做的微運動
- 圖解動作卡：頸部伸展、肩膀放鬆、手腕轉動、踝關節活動
- 每組 2-3 分鐘，不需要器材
- 久坐提醒：在醫院等候超過 1 小時時推送

#### 💌 心情小卡（觸發催產素 + 血清素）

- 每日隨機一張療癒插圖 + 溫暖文字
- 可收藏、分享給其他照護者（觸發催產素）
- 節日特別版（母親節、父親節等）
- 使用者可自己寫心情小語

#### ✨ 每日金句 + 感恩日記（觸發血清素）

- 精選照護者相關的勵志語錄，每天更新
- 感恩日記：每天寫三件感恩的事
- 可設定為手機桌布

#### 🏆 照護里程碑（觸發多巴胺 + 睪固酮）

- 照護天數計數器
- 完成待辦的累積徽章
- 呼吸練習連續天數
- 「你已經照顧家人 47 天了，了不起 🌿」

#### 🌱 快樂花園（核心養成系統）

使用者種一棵樹，每天完成快樂任務就獲得「陽光值」澆灌它，讓它從種子長成大樹。長成後解鎖下一棵，逐步建造自己的快樂森林。

**設計原則：架構完整，內容先克制。**
- 系統架構支援無限品種擴充（品種資料為 JSON config，新增品種 = 新增一筆資料）
- **POC 只放 3 棵樹**，刻意讓使用者不會一下子全養完
- 驗證機制是否被喜歡，再決定後續品種和寵物版擴充

**陽光值獲取規則：**

| 活動 | 陽光值 | 說明 |
|------|-------|------|
| 每日登入 | +1 ☀️ | 打開快樂區就算 |
| 完成呼吸練習 | +2 ☀️ | 任一呼吸法完成一輪 |
| 寫感恩日記 | +2 ☀️ | 寫下至少一件感恩的事 |
| 完成微運動 | +3 ☀️ | 跟著圖解做完一組 |
| 分享心情小卡 | +1 ☀️ | 分享給家人或朋友 |
| 完成小挑戰 🌶️ | +3 ☀️ | 微辣 |
| 完成小挑戰 🌶️🌶️ | +4 ☀️ | 中辣 |
| 完成小挑戰 🌶️🌶️🌶️ | +5 ☀️ | 大辣 |
| **每日上限** | **15 ☀️** | 防止一天刷爆，鼓勵「每天一點」 |

**成長階段（所有樹種共用）：**

| 階段 | 所需累積陽光 | 對應天數（約） | 視覺 |
|------|-----------|-------------|------|
| 🌰 種子 | 0 | 第 0 天 | 一顆種子在土裡 |
| 🌱 發芽 | 20 | 約 3-5 天 | 冒出小芽 |
| 🌿 幼苗 | 60 | 約 10-15 天 | 長出葉子 |
| 🪴 茁壯 | 120 | 約 21-30 天 | 枝葉茂密 |
| 🌳 大樹 | 200 | 約 40-50 天 | 開花結果，品種特有外觀 |

**POC 三棵樹（按解鎖順序）：**

| 順序 | 品種 | 解鎖條件 | 設計理由 |
|------|------|---------|---------|
| 1 | 🌸 櫻花樹 | 註冊即可開始 | 療癒感最強、辨識度高，第一印象 |
| 2 | 🌿 茶樹 | 第 1 棵長成後解鎖 | 照護者「泡杯茶休息」的意象 |
| 3 | 🍊 橘子樹 | 第 2 棵長成後解鎖 | 「結果」的隱喻，代表照護旅程的收穫 |

3 棵全養完約需 120-150 天（4-5 個月），正好覆蓋一般療程週期。全部養完後顯示：「更多品種即將到來 🌿」——不告訴使用者什麼時候，保留期待感。

**快樂森林（收集展示）：**
- 已養成的樹排列在一個橫向滾動的花園場景中
- 每棵樹記錄養成期間（起始日 - 完成日）
- 花園場景可截圖分享
- 花園隨樹的數量逐漸豐富（1 棵 = 空地 + 樹，2 棵 = 加小路，3 棵 = 加柵欄和蝴蝶）

**荒廢機制（溫和，不懲罰）：**

| 天數 | 表現 | 訊息 |
|------|------|------|
| 連續 3 天沒做 | 樹微微垂頭 | 「我有點渴了…」 |
| 連續 7 天沒做 | 樹打瞌睡 | 「沒關係，你先忙，我等你 🌿」 |
| 連續 14 天沒做 | 樹靜靜等待 | 「我還在這裡。」 |
| **永遠不會死亡或消失** | — | 照護者壓力已經夠大了，遊戲不該增加罪惡感 |
| 回來做任務 | 立刻恢復 + 歡迎動畫 | 「歡迎回來！☀️」 |

#### 🎯 舒適圈小挑戰（觸發腎上腺素，可控劑量）

- 每日一張小挑戰卡：「今天跟一個陌生人微笑」「走一條沒走過的路回家」
- 辣度分三級，獲得不同陽光值：

## 5. 系統架構

### 5.1 四層架構

① **輸入層** — 使用者上傳原始醫療文件（PDF、圖片、網頁、純文字）

② **AI 處理核心** — 前處理（OCR/爬蟲）→ LLM 結構化提取（Claude）→ 分類合併去重

③ **結構化資料層** — JSON 格式儲存（用藥、照護排程、注意事項），含版本控制

④ **輸出層** — 從同一份 JSON 生成多種格式（海報、行事曆、Checklist、藥品識別卡）

### 5.2 版本控制

以最新版為基礎，舊資料可追溯。版本對應療程時間軸（術後第 1 週、第 1 個月…）。

### 5.3 醫療安全

所有輸出標注「此內容由 AI 輔助整理，請與原始醫療文件核對」。

## 6. 技術棧

| 層級 | 選擇 | 理由 |
|------|------|------|
| 手機框架 | Expo (React Native) | 已會 React，學習成本最低；同時支援 Web 輸出供 SNS 試玩版 |

| AI 結構化 | Claude API (Sonnet) | 結構化提取 + 白話翻譯 |
| AI 生圖 | OpenAI DALL-E API | 海報加購功能 |
| 語音轉文字 | OpenAI Whisper API | Coming Soon（v0.2） |

## 7. 風險與緩解

| 風險 | 影響 | 緩解 |
|------|------|------|
| 醫療資訊正確性 — AI 可能搞錯劑量 | 嚴重 | 所有輸出加 disclaimer + 保留原始檔案連結 + 仿單專業版免費 |
| API 費用超過收入 | 中等 | 照護時段計費 + DALL-E 加購隔離 |
| 使用者不懂技術 | 中等 | 統一負擔 API，使用者零配置 |
| 換手機資料遺失 | 中等 | POC 已知風險，v0.2 規劃加密備份 |
| 濫用費用爆表 | 低 | 吃到飽 15 次上限 + SNS 版 email 控制 |
