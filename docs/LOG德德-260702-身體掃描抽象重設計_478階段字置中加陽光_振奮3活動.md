## 2. 決策紀錄

### 決策 1：身體掃描視覺 → 放棄人形，走「E 純光暈下行 + F 部位名稱」融合
- **最終方案**：一團柔光沿垂直淡軌道由上往下移動，經過某部位時該部位名稱放大發亮（過去的留淡、未到的更淡），到位輕震動。無人形、無方框。
- **原因**：花園樹已定案為「無臉抽象」，寫實/具象人形風格不一致且不夠療癒；鴿王先試過軟質色塊人形仍嫌「基本人形、不夠柔」，最後明確選 E+F。
- **替代方案**：①軟質色塊人形（否決：仍像生硬人形）②全抽象 6 區段方框（否決：太像清單）③Gemini 靜態人形（否決：寫實與抽象風打架、需外部產圖）。過程用 docs/carrius-bodyscan-concepts.html 出 6 案動態比較給鴿王挑。

### 決策 2：身體掃描底色 → 薰衣草紫漸層
- **最終方案**：底色由近白（#FFFDF8→#F1ECFD）改為薰衣草紫（#EAE3FB→#D6CBF3→#E0D6F7）＋中段柔光舞台；未到名稱透明度 .20→.24 配深底。
- **原因**：鴿王「要增加底色，不是全白」。各活動底色跟自己主色走（478/觀呼吸粉、身體掃描紫、微運動綠、曬太陽暖橘）。

### 決策 3：478 呼吸兩處
- **最終方案**：階段字（吸氣/屏息/吐氣）移進呼吸圓正中心、固定不隨縮放；大數字維持上方不疊圓。完成按鈕真的呼叫 completeActivity（+2☀）。
- **原因**：鴿王「把吐氣屏息放在圓中心」；且原按鈕寫了 +2 卻沒加，屬不一致。

### 決策 4：振奮 3 互動設計（鴿王 greenlit「照這個做」）
- **微運動**：跟做式，頸→肩→手臂→腳 4 動作各 20 秒，柔形脹縮輕擺，換動作輕震、走完 +3☀。
- **曬太陽**：沐浴式，暖陽光暈 90 秒緩緩漲亮，引導轉向光呼吸、每 3 秒暖脈動，+3☀。
- **寫個小目標**：輸入式（同感恩卡），寫一句存下來 +2☀。
- **原因**：三種既有互動模式（計時序列/環境計時/文字輸入）各取其一，全用 code 抽象視覺，不卡 Gemini。振奮組原 5 項砍成這 3（移除慶祝小成就、舒適圈小挑戰）。
